# VT Reader

テキスト、キャラクター画像、音声からリップシンク付きのキャラクター動画を作るローカルWebツールです。

## 起動

必要なものは Node.js 20以降と FFmpeg です。初回のみTypeScriptなどの開発依存をインストールします。

```bash
npm install
```

```bash
npm start
```

ブラウザで `http://localhost:4173` を開きます。

## 使い方

1. 「キャラクター」で基本の立ち絵を選びます。透過PNG推奨です。
2. 台本を入力します。`[happy]`、`[sad]`、`[angry]`、`[surprised]`、`[neutral]` で表情を指定できます。
3. 音声ファイルを選ぶか、Irodori-TTSまたはVOICEVOXから音声を生成します。
4. プレビューで口の位置と大きさを調整し、「MP4を書き出す」を押します。

表情差分画像は任意です。未設定の表情では基本画像が使われます。音声なしの場合は文字数から尺を推定し、無音の動画を書き出します。

## VOICEVOX連携

VOICEVOX Engineを起動してからVT Readerを立ち上げます。既定の接続先は `http://127.0.0.1:50021` です。別ホストの場合は次のように指定します。

```bash
VOICEVOX_URL=http://localhost:50021 npm start
```

## Irodori-TTS連携

高品質な小説朗読には、公式のOpenAI互換サーバーを別プロセスで起動します。RTX 5080 LaptopではCUDA 12.8バックエンドを使用します。

```bash
git clone https://github.com/Aratako/Irodori-TTS-Server.git ../Irodori-TTS-Server
cd ../Irodori-TTS-Server
uv sync --extra cu128
cp .env.example .env
```

`.env`では次の設定がおすすめです。

```env
IRODORI_TTS_BACKEND=cu128
IRODORI_MODEL_DEVICE=cuda
IRODORI_MODEL_PRECISION=bf16
IRODORI_PRELOAD=true
```

サーバーを起動します。初回起動時はモデルとコーデックが自動取得されます。

```bash
uv run --no-sync python -m irodori_openai_tts --host 127.0.0.1 --port 8088
```

セットアップ後は、VT ReaderとIrodori-TTS Serverをプロジェクト直下のシェルでまとめて起動できます。

```bash
./start-with-irodori.sh
```

停止・再起動・状態確認は、別のターミナルから次のように実行できます。

```bash
./start-with-irodori.sh stop
./start-with-irodori.sh restart
./start-with-irodori.sh status
```

またはnpm経由でも起動できます。

```bash
npm run start:irodori
```

引数なしまたは`start`で起動します。`Ctrl+C`でも、このシェルが起動した両方のプロセスを終了します。すでに外部プロセスとして起動していたIrodori-TTS Serverは、`stop`の対象になりません。公式サーバーを別の場所へ配置した場合は、`IRODORI_SERVER_DIR=/path/to/Irodori-TTS-Server ./start-with-irodori.sh`で指定できます。

統合起動ではVoiceDesignモデルと、VoiceDesignで生成した落ち着いた朗読用の`assets/voices/calm_cute_female.wav`を既定で使用します。別のモデルや音声を使う場合は、`IRODORI_HF_CHECKPOINT`、`IRODORI_VOICES_DIR`、`IRODORI_DEFAULT_VOICE`を環境変数で上書きできます。
初回のモデル取得・読込は最大15分待機します。必要なら`IRODORI_STARTUP_TIMEOUT=1200 ./start-with-irodori.sh`のように秒数を変更できます。

別の接続先を使用する場合は、VT Reader起動時に指定します。

```bash
IRODORI_TTS_URL=http://127.0.0.1:8088 npm start
```

音声タブでIrodori-TTSを選択すると、声ID、速度、VoiceDesign用の話し方を設定できます。参照音声は同じ画面から公式サーバーへ登録できます。本人の許可を得た音声、または自分が権利を持つ音声だけを使用してください。

音声生成は音声タブだけでなく、台本タブの「音声を生成」ボタンからも実行できます。台本タブから生成した場合も、音声タブで選択しているエンジン・声・速度・品質設定が使用されます。

台本タブのタイトル・著者を入力すると、生成音声の本文より前へ「作品名」「著者」として自動追加されます。青空文庫から取得した場合は、作品ページのタイトル・著者も可能な範囲で自動入力します。

「動画冒頭のテキスト」「動画最後のテキスト」には、朗読の前後へ表示するタイトルカードを入力できます。空欄ならカードは追加されません。表示秒数はそれぞれ1〜20秒で指定でき、カード表示中もBGMと環境音は流れます。再生順は「冒頭カード → タイトル・著者と本文の朗読 → 終了カード」です。

上部の「プロジェクト保存」では、台本・作品情報・タイトルカード・字幕・音声設定・BGM／環境音・キャラクター調整値・レイヤー位置を`.roudoku.json`へ保存できます。「プロジェクト読込」で復元できます。旧形式の`.vtreader.json`も引き続き読み込めます。生成済み朗読音声や選択したカスタム画像・カスタム音源の実データはJSONへ埋め込まれないため、読込後に音声を再生成し、外部素材は必要に応じて再選択してください。

誤読した漢字は、台本下の「漢字の読み修正」へ1行ずつ`漢字=よみ`と入力し、「台本へルビを一括反映」を押します。同じ語は台本内の全箇所へ反映されます。文脈で読みが違う箇所は、台本へ直接`｜漢字《よみ》`と指定してください。元音声の字幕タイミング補正が完了した後に「変更文だけ音声を修正」を押すと、読みが変わった文だけを現在の音声設定で生成し、元WAVの該当区間へまとめて差し替えます。既に同じルビが付いている場合も、読み修正欄に対象語を入力してこのボタンを押せば、その語を含む文を同じルビで強制的に再生成できます。読み修正一覧と反映済みルビはプロジェクトへ保存されます。

台本タブの「朗読中の文章を画面に表示」を有効にすると、現在読んでいる文章がCanvasの最前面へ表示されます。字幕は風に運ばれるような横移動・残像・フェード付きで切り替わり、プレビューだけでなくMP4にも記録されます。
字幕エフェクトは「風フェード」と「黒文字・斜めツイスト」から選択できます。斜めツイストでは、左右が溶ける薄い生成り色のフェード背景に、少し古風な日本語明朝体の黒文字が一文字ずつ時間差で斜め方向から現れます。各文字へ個別に回転・傾き・横幅変形をかけ、文字自体がひねられながら正面を向く演出です。文字の外枠は付けません。文字サイズ・左端位置・縦位置は台本タブのスライダーで調整できます。横位置は字幕の中央ではなく左端を基準とし、右端までの幅で長文を自動改行します。

Irodori-TTSの生成品質は、高音質（56ステップ）・標準（40ステップ）・高速確認（24ステップ）から選べます。デフォルトは高音質です。句読点ごとに音声を分け、読点・句点・感嘆符・疑問符・改行に応じた無音を音声へ挿入するため、モデルが句読点を無視した場合も間が保持されます。句点（`。`・`.`）の無音は1.04秒です。

音声タブではBGMも設定できます。同梱サンプルまたは任意の音声ファイルを選び、音量とループを調整できます。BGMは朗読音声と別に再生されるためリップシンクには影響せず、MP4書き出し時には自動的にミックスされます。

環境音もBGMとは独立したレイヤーとして設定できます。同梱の「風」「田園風景」または任意の音声ファイルを選択でき、音量とループを個別に調整できます。デフォルトは「風」12%です。朗読音声・BGM・環境音はプレビューとMP4で同時にミックスされます。

VoiceDesignを使う場合は、Irodori-TTS Server側のチェックポイントを`Aratako/Irodori-TTS-600M-v3-VoiceDesign`へ変更します。通常の500M v3モデルでは話し方欄が無視されます。

## テスト

```bash
npm run typecheck
npm test
```

TypeScriptの原本は `src/`、コンパイル結果は `dist/` に生成されます。`npm start` はビルド後に `dist/server.js` を起動します。

動画はブラウザでWebMとして録画した後、サーバー側のFFmpegでH.264/AACのMP4へ変換します。画像・音声・動画は外部サービスへ送信されません。
